# Agent 自动维护规则

## 规则 P-1: 项目数据同步

当 Agent 操作了任何毕业设计项目（`D:\大学生运动打卡...` 等），结束后必须同步更新对应 `项目总控/*.md` 文件的状态字段。

## 规则 P-2: 新项目自动登记

当 Agent 新建或导入新项目到 `D:\` 下，必须:
1. 在 `项目总控/` 下创建对应的 `.md` 文件
2. 包含完整的 YAML frontmatter
3. 更新 `项目总控/_项目总控库.md` 的链接列表

## 规则 K-1: 知识库编译触发

听到 "编译知识库"、"compile wiki"、"更新知识库" → 执行 `_知识库迭代机制.md` 第二章的增量编译流程:
1. 扫描 `原料/`
2. 提取概念
3. 匹配现有文章
4. 增量更新或新建

## 规则 K-2: 知识库审计触发

听到 "审计知识库"、"健康检查" → 执行 `_知识库迭代机制.md` 第五章的五项检查:
1. 一致性
2. 完整性
3. 新鲜度
4. 连接发现
5. 漏洞扫描

结果写入 `_迭代日志.md`。

## 规则 K-3: Q&A 输出归档

Agent 基于知识库完成的每次复杂问答，结束后自动将结果归档到 `知识库/输出/分析报告/`，并回链到被引用的知识文章。

## 知识库规模里程碑

| 阶段 | 目标 | 状态 |
|:---|:---|:---|
| 当前 (第 2 次迭代后) | 21 篇文章 / ~12,000 词 | ✅ 已达成 |
| Phase 1 | 50 篇文章 | ⏳ 积累中 |
| Phase 2 | 100 篇文章 / ~40 万词 | 🔒 远期 |
| Phase 3 | 合成数据 + QA 对生成 | 🔒 远期 (100+ 篇后) |
| Phase 4 | 微调专属模型 | 🔒 远期 (200+ 篇后) |
